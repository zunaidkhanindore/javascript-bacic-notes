Lecture 12 – Async JavaScript (Promises, fetch, async/await)

Transcript Summary

  • JS single-threaded है; async tasks Promise-based APIs से handle होते हैं।
  • Promise states: pending → fulfilled/rejected.
  • fetch network requests के लिए; async/await syntax sugar on Promises.
  • Error handling: .catch या try...catch.

Promise Basics

const p = new Promise((resolve, reject) => {
  setTimeout(() => resolve("done"), 500);
});
p.then(v => console.log(v)).catch(err => console.error(err));

fetch + async/await

async function loadUsers() {
  try {
    const res = await fetch("https://jsonplaceholder.typicode.com/users");
    if (!res.ok) throw new Error("Network error: " + res.status);
    const data = await res.json();
    console.log("Users:", data.map(u => u.name));
  } catch (err) {
    console.error("Failed:", err.message);
  } finally {
    console.log("Always runs");
  }
}
loadUsers();

Parallel vs Sequential

PatternSnippetUse When
Parallel const [a,b] = await Promise.all([fa(), fb()]) Independent tasks
Sequential const a = await fa(); const b = await fb(a); Dependent tasks
